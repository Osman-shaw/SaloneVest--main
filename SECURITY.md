# SaloneVest Security Policy

## Overview

SaloneVest is a non-custodial blockchain application. Security is a shared responsibility between users and the platform.

## Non-Custodial Model

- **Your Keys, Your Control**: Private keys never leave user's Phantom wallet
- **No Access**: SaloneVest cannot access or recover user funds
- **Blockchain Settlement**: All transactions finalized on Solana

## Security Best Practices for Users

### 1. Wallet Security

- Never share seed phrase or private keys
- Use strong, unique passwords
- Enable wallet backup and recovery options
- Keep Phantom extension updated

### 2. Transaction Verification

- Always verify transaction amounts before signing
- Check recipient addresses carefully
- Review fee estimates
- Confirm on blockchain explorer

### 3. Account Protection

- Use unique email for SaloneVest account
- Enable 2FA on email account
- Monitor account activity regularly
- Review transaction history

## Platform Security

### 1. Data Protection

- HTTPS/TLS encryption in transit
- No sensitive data stored server-side
- User data encrypted at rest
- Regular security audits

### 2. Smart Contract Security

- Solana program audited by third party
- Program Derived Addresses (PDAs) for transparency
- Source code open-source on GitHub
- Upgrade authority controlled

### 3. Infrastructure Security

- Vercel enterprise security
- DDoS protection enabled
- Rate limiting on endpoints
- Web Application Firewall

## Vulnerability Disclosure

### Responsible Disclosure

If you discover a security vulnerability:

1. **Email**: security@saloneVest.io (encrypted preferred)
2. **Do NOT**: Post publicly or exploit vulnerability
3. **Include**: Detailed description and proof of concept
4. **Wait**: Maximum 90 days for patch release

### Response Timeline

- **24 hours**: Acknowledgment
- **7 days**: Initial assessment
- **30 days**: Patch development
- **45 days**: Public disclosure after patch
